Test table layout options. We primarily interested here in vertical layouts, so the table rows generally only have one cell per row.

This is the type of layout we were trying for. We want to use absolute:position so that the contents fill the parent. We want the title to automatically grow (vertically) to accomodate the font selected. We want the main content to fill the remaining space (less the header) and to scroll its content vertically.

Unfortunately, this approach requires that the designer explicitly match the font and padding of the header with the absolute layout position of the content area. See the style="top:24px" in the example below.

The hope had been to find a way to combine tables with absolute positioning to resolve this problem. After both research and various experiments, it appears that there is simply no way to accomplish this layout automatically via tables. For now, we will hardcode the layout positions into the css.
